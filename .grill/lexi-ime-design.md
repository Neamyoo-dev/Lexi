# Grill: Lexi 输入法方案设计
Date: 2026-06-01

## Intent
开发一款公开发布给中文用户的输入法，核心差异是「极致视觉美感 + 极简轻量」，让用户因为设计和体验主动从搜狗/百度/讯飞/微信键盘切换过来。

## Constraints
- 已使用 Tauri + Rust（Vanilla HTML/CSS/JS）创建项目
- 没有 macOS 开发环境，先 Windows，再 macOS，最后移动端
- 1-2 个月出 MVP（最小可用版本）

## Key decisions
- **平台路线**: Windows 先发 → macOS → 移动端（分阶段）
- **核心技术**: Tauri + Rust 架构，Rust 处理 TSF 和 RIME 集成，Tauri WebView 渲染候选栏 UI
- **前端框架**: Vanilla HTML/CSS/JS（无框架依赖）
- **输入方案**: 仅拼音，未来可通过插件/扩展支持其他方案
- **输入引擎**: 基于 RIME（中州韵）做壳，不自研引擎
- **候选栏风格**: 传统横排跟随光标，毛玻璃 + 纯色极简融合风格
- **候选词布局**: 默认 6 个，可调节；数字键选词 + Tab/[] 翻页
- **选中高亮**: 底部条样式
- **主题系统**: 内置 2 套主题（浅色 1 套 + 深色 1 套），提供调色板和渐变开关
- **中英文切换**: Shift 为主，可选 CapsLock
- **词库策略**: 预装高质量词库（如朙月拼音 + 八股词库），开箱即用
- **同步方案**: 本地文件手动导入导出，零后端
- **特殊输入 v1**: v 模式（数字/日期/计算）+ 中英文混合输入 + 表情快捷输入（符号面板延迟实现）
- **设置项**: 可调节候选词数量、主题/颜色自定义、中英文切换方式等

## Surfaced assumptions
- 用户对 Windows TSF 输入法框架的架构细节需要额外梳理
- RIME 集成需要 Rust 绑定或 C FFI
- 候选栏的 Tauri 窗口需要以特殊方式（无边框、置顶、跟随光标）呈现，不是标准应用窗口

## Open questions
- 具体的 RIME Rust 绑定方案（rime.rs / librime 直接 FFI）
- TSF DLL 是否以单独的 Rust cdylib 项目存在，还是与 Tauri 应用共用
- Windows 安装包打包策略（MSI 安装时注册 TSF）
- 命名 "Lexi" 是否用于正式发布，还是只是项目代号

## Out of scope
- 五笔/双拼等非拼音输入方案（第一版不实现）
- 云服务/账号同步（仅本地）
- 第三方主题系统（仅内置主题）
- U 模式拆字输入
- 符号面板（延迟到功能完善后）
